# SKILLS 04-09 — Reference Guide

## Multi-LLM provider — how it works in these skills

Every agent in Skills 04-09 calls `call_llm()` from `app.core.llm_provider`.
**Never import openai, anthropic, or any provider SDK directly in agent files.**

```python
# ✓ CORRECT — every agent does this
from app.core.llm_provider import call_llm

response_text = await call_llm(
    messages=[{"role": "user", "content": prompt}],
    temperature=0.0,         # 0.0 for deterministic compliance decisions
    response_format={"type": "json_object"},
)

# ✗ WRONG — never do this in agent files
from openai import AsyncOpenAI  # hardcodes provider
client = AsyncOpenAI(...)
```

Changing `LLM_PROVIDER=anthropic` in `.env` makes ALL agents use Claude instantly.
No agent code changes. This is the config-driven architecture in practice.

**Provider capabilities by agent:**

| Agent | Temperature | Why |
|---|---|---|
| Planner | 0.1 | Needs slight variation for task decomposition |
| Extraction | 0.0 | Must be deterministic — same doc → same facts |
| Evaluation | 0.0 | Compliance is binary — no temperature variation |
| Comparator | 0.0 | Rankings must be stable across runs |
| Decision | 0.0 | Governance decisions must be repeatable |
| Explanation | 0.2 | Report prose benefits from slight variation |
| Critic | 0.0 | Validation must be deterministic |

**Note on Anthropic Claude:** Claude does not support `response_format={"type": "json_object"}`.
For Anthropic provider, end the prompt with "Respond with only valid JSON, no prose."
`call_llm()` handles this automatically — see `app/core/llm_provider.py`.

# SKILL 04 — Extraction Agent + Structured Fact Store
**Builds:** app/agents/extraction.py
**Output:** ExtractionOutput with grounding_quote verified programmatically
**Key rule:** Every extracted value must have grounding_quote that appears verbatim in source text

## What the Extraction Agent does

Reads retrieved chunks from the Retrieval Agent.
Extracts typed facts: certifications, insurance, SLAs, projects, pricing.
Stores immediately in PostgreSQL via app/db/fact_store.py.
Every fact has a grounding_quote — the exact text from the source chunk.
The Critic verifies grounding_quote appears verbatim in source text.
If it does not appear: hard flag, hallucination confirmed.

## Key implementation pattern

```python
# app/agents/extraction.py — core pattern

async def run_extraction_agent(
    retrieval_output: RetrievalOutput,
    vendor_id: str,
    org_id: str,
    agent_config: dict
) -> tuple[ExtractionOutput, object]:

    # Build context from retrieved chunks
    source_chunks = {
        chunk.chunk_id: chunk.text
        for chunk in retrieval_output.chunks
    }
    context = "\n\n---\n\n".join(
        f"[{chunk.chunk_id}]\n{chunk.text}"
        for chunk in retrieval_output.chunks
    )

    # Ask GPT-4o to extract structured facts
    # IMPORTANT: prompt requires grounding_quote for every value
    response = await call_openai_with_backoff(
        client,
        model=settings.openai_model,
        temperature=0.0,
        response_format={"type": "json_object"},
        messages=[
            {
                "role": "system",
                "content": """Extract structured facts. For EVERY value you extract,
you MUST provide the exact quote from the source text in grounding_quote.
If you cannot find a verbatim quote for a value, set the value to null.
Never invent values. Return only what is explicitly stated.

Return JSON matching ExtractionOutput schema.
Respond with only valid JSON, no prose or markdown."""
            },
            {
                "role": "user",
                "content": f"Extract from:\n\n{context}"
            }
        ]
    )

    # Parse response into Pydantic models
    # Critic then verifies ALL grounding_quotes programmatically
    extraction = ExtractionOutput(**json.loads(...))

    # Run Critic — programmatic grounding check
    critic = critic_after_extraction(extraction, source_chunks)

    # Save to PostgreSQL if Critic approved
    if critic.overall_verdict != CriticVerdict.BLOCKED:
        save_extraction_output(extraction, doc_id)

    return extraction, critic
```

# SKILL 05 — Evaluation Agent + Comparator Agent
**Builds:** app/agents/evaluation.py, app/agents/comparator.py
**Output:** EvaluationOutput, ComparatorOutput
**Key rule:** Evaluation Agent reads PostgreSQL facts — NOT Qdrant chunks

## What makes Evaluation Agent different

It reads structured facts from PostgreSQL, not raw text from Qdrant.
This is what prevents LLM scoring inconsistency.
The LLM receives typed data: status="current", amount_gbp=5000000
Not raw prose: "we maintain professional indemnity insurance of five million pounds"

```python
# app/agents/evaluation.py — core pattern

async def run_evaluation_agent(
    vendor_id: str,
    org_id: str,
    agent_config: dict
) -> tuple[EvaluationOutput, object]:

    # Get pre-extracted structured facts from PostgreSQL
    # NOT from Qdrant — that is the key difference
    facts = get_vendor_facts(org_id, vendor_id)

    # For each mandatory check, evaluate against typed facts
    compliance_decisions = []
    for check in mandatory_checks:
        decision = await _evaluate_mandatory_check(
            check=check,

    criterion_scores = []
    for criterion in scoring_criteria:
        score = await _score_criterion(
            criterion=criterion,
            facts=facts,
            config=agent_config
        )
        criterion_scores.append(score)

    output = EvaluationOutput(...)
    critic = critic_after_evaluation(output, extraction_output)
```

## Decision Agent

Reads EvaluationOutput and ComparatorOutput.
Applies governance config — approval tier routing.
Produces RejectionNotice with verbatim evidence_citations.
Hard block if any rejection has empty evidence_citations.
Hard block if all vendors rejected — escalates for human review.

```python
# app/agents/decision.py — approval tier routing

def route_to_approval_tier(
    contract_value: float,
    agent_config: dict
) -> ApprovalRouting:
    tiers = agent_config["governance"]["approval_tiers"]
    tiers_sorted = sorted(tiers, key=lambda t: t["max_value"] or float("inf"))

    for tier in tiers_sorted:
        max_val = tier.get("max_value")
        if max_val is None or contract_value <= max_val:
            sla_deadline = datetime.utcnow() + timedelta(
                hours=tier["sla_hours"]
            )
            return ApprovalRouting(
                approval_tier=tier["tier"],
                approver_role=tier["approver_role"],
                contract_value=contract_value,
                sla_hours=tier["sla_hours"],
                sla_deadline=sla_deadline
            )
```

## Explanation Agent

Every claim goes through grounding check before inclusion.
Ungrounded claims are removed and counted in ungrounded_claims_removed.
If grounding_completeness < 0.70: Critic hard blocks.

```python
# app/agents/explanation.py — grounding verification

def verify_grounding(
    claim_text: str,
    grounding_quote: str,
    source_chunk_id: str,
    source_chunks: dict[str, str]
) -> bool:
    """
    Programmatic check — not LLM.
    The grounding_quote must appear verbatim in the source chunk.
    """
    source = source_chunks.get(source_chunk_id, "")
    if not source or not grounding_quote:
        return False
    return grounding_quote.strip() in source
```

## LangFuse integration

```python
# app/core/langfuse_client.py
# ═══════════════════════════════════════════════════════
# IMPORTANT: langfuse v4 SDK (released March 2026) is a
# complete rewrite. The v2.x API no longer works.
# Read migration guide: https://langfuse.com/docs/sdk/python
# ═══════════════════════════════════════════════════════

from langfuse import Langfuse
from langfuse.decorators import observe, langfuse_context
from app.config import settings

_lf = None


def get_langfuse() -> Langfuse:
    global _lf
    if _lf is None:
        _lf = Langfuse(
            public_key=settings.langfuse_public_key,
            secret_key=settings.langfuse_secret_key,
            host=settings.langfuse_host
        )
    return _lf


def log_evaluation_run(
    run_id: str,
    agent_name: str,
    input_data: dict,
    output_data: dict,
    critic_verdict: str,
    latency_ms: int,
    org_id: str
):
    """Log an agent run as a LangFuse trace (v4 SDK pattern)."""
    lf = get_langfuse()
    # v4 uses trace() same as v2 for basic tracing
    trace = lf.trace(
        name=f"{agent_name}_run",
        id=run_id,
        metadata={
            "org_id": org_id,
            "critic_verdict": critic_verdict,
            "latency_ms": latency_ms
        },
        input=input_data,
        output=output_data
    )
    # v4 change: must call flush() explicitly or use context manager
    lf.flush()
    return trace


def log_critic_flag(
    run_id: str,
    flag_severity: str,
    flag_check: str,
    agent: str,
    org_id: str
):
    """
    Critical for monitoring — track flag rates over time.
    Alert if hard_flag_rate > 5% across evaluations.
    v4 SDK: score() method signature unchanged.
    """
    lf = get_langfuse()
    lf.score(
        name=f"critic_flag_{flag_severity}",
        trace_id=run_id,
        value=1.0,
        comment=f"{agent}: {flag_check}"
    )
    lf.flush()
```
